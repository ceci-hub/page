2. What is the difference between a list, vector, and data frame?

💡
  • R: vector, list, data.frame, tibble
  • Python: list, dict,

DataFrame When to use each and why performance matters

R
💡R

Vector A vector is the most basic data structure in R. It is one‑dimensional and homogeneous, meaning all elements must be of the same type (numeric, character, logical, etc.). When to use:

Simple numeric or logical operations Mathematical calculations Statistical functions (means, sums, transformations)

Why performance matters: Vectors are highly optimized in R. Vectorized operations avoid explicit loops and are much faster and more memory‑efficient than iterative approaches.

List A list is a heterogeneous container that can hold elements of different types and sizes, including vectors, data frames, models, and even other lists. When to use:

Storing complex objects (e.g., model outputs, nested results) Returning multiple objects from a function Handling hierarchical or nested data

Why performance matters: Lists are flexible but less efficient for computation. They are best used for organization, not for numeric-heavy operations.

Data Frame A data frame is a tabular structure where:

Columns are vectors Each column is homogeneous Different columns can have different types

When to use:

Traditional data analysis Regression modeling Aggregation and joins

Why performance matters: Data frames are intuitive but can be slower with very large data due to copying behavior and less strict column handling.

Tibble A tibble is a modern variant of a data frame (from the tidyverse) with stricter and more predictable behavior. When to use:

Tidyverse workflows (dplyr, ggplot2) Interactive analysis Large or complex datasets

Why performance matters: Tibbles avoid partial matching and automatic type changes, reducing bugs. Performance is similar to data frames, but developer productivity and correctness are significantly improved.

Python
💡Py

List A Python list is an ordered, mutable collection that can hold mixed data types. When to use:

Small collections Iterative processing General-purpose storage

Why performance matters: Lists are flexible but slow for large numerical computations. Iterating over lists in Python is expensive compared to vectorized alternatives.

Dictionary (dict) A dictionary stores key–value pairs with fast lookup. When to use:

Mapping relationships Storing metadata Fast lookups by key

Why performance matters: Dictionaries offer O(1) average lookup time, making them ideal for indexing and caching—but not for numeric computation.

DataFrame (pandas) A DataFrame is a 2‑dimensional, labeled tabular structure, similar to an R data frame or SQL table. When to use:

Data cleaning and transformation Feature engineering Exploratory analysis Modeling pipelines

Why performance matters: pandas DataFrames use vectorized operations built on NumPy, which are far more efficient than Python loops. However, very large datasets may still require chunking or distributed tools (e.g., Dask, Spark).

10. Describe a reproducible data pipeline you have built. How did you structure it, and how would another person rerun it?

💡

What a strong answer should include:

  • A clear entry point (master script, main file, or workflow)

  • Separation of steps (cleaning, transformation, analysis)

  • How paths, environments, or configurations are handled

  • How outputs (tables, models, reports) are generated automatically

  • How someone else could rerun the pipeline with minimal effort
